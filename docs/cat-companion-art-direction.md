# 小猫桌面宠物美术方案

## 角色定位

第一版主角从占位小狗切换为小猫，只支持单一小猫角色，不做复杂多宠物系统。整体方向是“高清卡通贴纸风”，重点表现它趴在屏幕边缘陪伴用户，而不是传统 App 内的养成宠物。

## 视觉规范

- 风格：高清卡通贴纸风，软萌、干净、治愈，可略微活泼。
- 背景：所有精灵帧必须是透明 PNG，不允许蓝底、白底、棋盘格、圆角卡片、投影边框。
- 轮廓：保留粗黑描边，保证缩放到 128px 时仍然清楚。
- 边缘感：猫的爪子、头部和尾巴要表达“贴在屏幕边缘”。屏幕边缘线可以作为动作构图的一部分，但后续更推荐拆成独立程序绘制层。
- 尺寸：源图可以较大，项目内运行素材建议保存为 256x256 PNG，由程序缩放到 128/160。

## 第一批动作

当前只生成 4 个核心动作的概念帧：

| 动作 ID | 用途 | 后续建议帧数 |
| --- | --- | --- |
| `idle_lie` | 默认趴边状态 | 6-8 |
| `sleep` | 用户空闲或深夜时睡觉 | 4-6 |
| `peek` | 用户回来、轻互动、探头 | 6-10 |
| `nudge` | 工作过久后的轻提醒 | 6-10 |

其余动作仍沿用现有动作表，后续确认角色风格后再扩展：

- `tail_wag`
- `wake_up`
- `stretch`
- `look_around`
- `concerned`
- `happy_bounce`
- `walk_edge`
- `sit_wait`

## 贴边交互方向

当前窗口会根据拖拽后吸附的边缘切换基础姿态：

- 底部边缘：`idle_lie`，表现为趴在任务栏/屏幕底边上。
- 左右边缘：`peek`，表现为从屏幕侧边探头。
- 远离底部的侧边位置：后续应扩展为侧边站立、沿边行走或攀爬动作；当前先用 `peek` 作为占位。

### 贴边锚点规则

程序现在不再只把窗口外框贴到屏幕边缘，而是使用精灵锚点把“素材内部的屏幕边缘线”对齐到真实屏幕边缘：

- `idle_lie` / 底边：默认以精灵高度 100% 的位置作为底边趴靠线，确保小猫完整显示在任务栏上方，不伸进任务栏区域。
- `peek` / 右边：默认以精灵宽度 75% 的位置作为右边探头线，允许约 25% 宽度越出屏幕。
- `peek` / 左边：默认以精灵宽度 25% 的位置作为左边探头线，允许约 25% 宽度越出屏幕。
- 顶边：暂时不作为主体验方向，当前保留基础吸附能力。

这意味着后续素材不能只保证透明裁剪正确，还要保证构图里的“扒住边缘线”落在对应锚点附近。底边素材不要自带黑色横线；真实边缘由 Windows 任务栏/屏幕边界提供。否则程序虽然已经贴边，视觉上仍会像浮在边框外。

下一批建议新增或细化的素材：

| 方向 | 建议动作 | 说明 |
| --- | --- | --- |
| 右边框探头 | `peek` 侧边版本 | 爪子扒住右边缘，头从屏幕外探出 |
| 沿边移动 | `walk_edge` | 在底边或侧边移动，不应长期回退成趴着 |
| 自主睡觉 | `sleep` 多帧 | 走累后趴下睡，身体轻微呼吸 |
| 自主玩耍 | `happy_bounce` 或新增玩耍动作 | 小幅扑爪、摆尾、翻身，避免打扰用户 |

## AI 控制预留

第一版不开放复杂动作 API，只保留明确的动作 ID。后续 AI 可以根据隐私摘要和宠物性格选择动作、表情和文案，但 AI 不应直接拿到窗口标题、应用名、文件名等隐私信息。

默认性格仍为 `gentle`。后续可以开放用户选择，例如温柔陪伴、活泼撒娇、安静守护。
